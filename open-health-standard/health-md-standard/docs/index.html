Healthcare Data Deserves Better Standards

EIR Format defines an open YAML-based standard for structuring healthcare information optimized for Large Language Models while preserving privacy, accuracy, and clinical context. Load directly into AI conversations via MCP servers.

See EIR Format in Action

Here's a sample EIR file showing how healthcare data is structured in YAML:

# Electronic health Information Record (EIR) Format metadata: format_version: "1.0" privacy_level: "anonymous" record_id: "anonymous-001" patient: name: "Anonymous Patient 001" age: 34 sex: "female" occupation: category: "Technology" current_medications: - name: "Metformin" strength: "500mg" dosage: "500mg twice daily" indication: condition: "Type 2 Diabetes Mellitus" icd_10: "E11.9" started: "2024-01-15" clinical_notes: "Well tolerated, excellent control" lab_results: glucose_metabolism: - test_name: "Hemoglobin A1C" results: - date: "2024-02-10" value: 6.8 unit: "%" reference_range: "<7.0" trend: "decreasing"
